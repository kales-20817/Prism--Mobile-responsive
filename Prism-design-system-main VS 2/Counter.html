Counter

The Counter is a compact numeric indicator used to display counts, badges, or notification numbers. Counters are pill-shaped elements that adapt their color and intensity based on semantic state.

Number ValueNumeric text. Defaults to "57". Font: Zoho Puvi Medium, centered horizontally and vertically.

Content GuidelinesRecommended

Do
  • Keep numeric values short (1-3 digits, show "99+" for large numbers).
  • Use Subtle style for inline contexts where the counter is secondary information.
  • Use Intense for high-visibility scenarios (e.g., unread notifications).
  • Match state color to semantic meaning (Critical for errors, Success for completions).
Don't
  • Don't use counters for non-numeric content (use Badge instead).
  • Don't use as a standalone status indicator without context.
  • Don't mix Subtle and Intense counters in the same row or group.
  • Don't use very large numbers without truncation (e.g., "1542" should be "99+").

When and Where to Use

Counters communicate numeric quantities at a glance, always paired with contextual labels or parent components.

When to use

  • Display unread notification counts on navigation items.
  • Show item counts in tabs, filters, or category labels.
  • Indicate quantity in list rows or table cells.
  • Display status-grouped counts in dashboards.

When NOT to use

  • For non-numeric content (use Badge instead).
  • As a standalone status indicator (use Status Indicator).
  • For categorization labels (use Tag).
  • When count meaning is unclear without context.

Semantic States

Each state maps to a specific semantic meaning, ensuring consistent communication across the interface.

  • Active: Primary or informational counts such as unread messages, new items, or selected filters.
  • Critical: Error or failure counts indicating items requiring immediate attention (e.g., failed deployments).
  • Moderate: Warning or alert counts for situations needing review but not yet critical.
  • Important: Urgent but non-critical counts, such as overdue tasks or pending approvals.
  • Success: Completed or positive counts, like resolved tickets or successful operations.
  • Default: Neutral or secondary counts for general-purpose use without semantic weight.
  • Acknowledge: Acknowledged or informational-secondary counts, often used for dismissed or archived items.

Complete Do's and Don'ts

Visual and interaction guidelines to ensure counters are effective, consistent, and accessible.

Do
  • Keep values short: Use 1-3 digits. Display "99+" for values exceeding 99.
  • Provide context: Always pair counters with a parent label, icon, or navigation item.
  • Consistent style per group: Use either all Subtle or all Intense within a single UI row.
  • Match semantics: Use Critical for errors, Success for completions, Active for informational.
Don't
  • Non-numeric content: Don't display text or icons inside a Counter (use Badge instead).
  • Standalone use: Don't use a Counter without adjacent context explaining what is being counted.
  • Mixed styles: Don't mix Subtle and Intense in the same row or contextual group.
  • Large numbers: Don't display raw large numbers like "1542" without truncation.

Accessibility

        
Guidelines for ensuring Counter components are perceivable and understandable across all assistive technologies.


        

          
Accessibility Guidelines

          

            
GuidelineRequirementDetails

            
              

                aria-label
                Required
                Provide full context, e.g., aria-label="5 unread messages" rather than just "5". The numeric value alone is insufficient for screen reader users.
              

              

                Color Contrast
                WCAG AA
                All Subtle and Intense combinations meet WCAG AA contrast minimums (4.5:1 for text). The Moderate/Intense variant uses black text (#000000) on yellow (#F9B21D) to maintain adequate contrast.
              

              

                Screen Reader
                Contextual announcement
                The count should be announced with context from the parent element. Use aria-describedby or wrap the counter within a labeled container so the relationship is clear.
              

              

                Color Independence
                Required
                Do not rely solely on color to convey the semantic state of a counter. Always provide additional context through labels, position, or descriptive text so that users who cannot perceive color differences still understand the meaning.
