Typography

Use typography to create hierarchy, improve scan speed, and support confident decision-making across endpoint workflows.

In Prism, typography structures dashboards, device inventory, alert queues, policy builders, patch workflows, AI summaries, and remediation confirmations.

A strong typography system helps admins understand state, risk, and action priority faster in information-dense environments.

Typeface system

Prism supports three UI typeface modes through the Font-family variable collection in Figma. Switching modes updates every text element across the design system automatically.

Zoho Puvi

Default mode

Zoho Puvi is the primary UI typeface for Prism.

Family: Zoho Puvi Weights: Regular, Medium, SemiBold, Bold Mono companion: Zoho Puvi Mono

Use Zoho Puvi for:

  • Page titles and section headings
  • Widget titles and body copy
  • Labels, buttons, and navigation
  • Table headers and metadata
  • AI insights and severity badges

Why Zoho Puvi

  • Strong readability at 12-14px
  • Clean enterprise personality
  • Excellent multilingual support
  • Better numeral clarity for tables
  • Strong consistency with Zoho ecosystem products

Lato

Alternate mode

Lato is the first alternate typeface for Prism. A humanist sans-serif with a warm, approachable feel that maintains excellent readability across UI densities.

Family: Lato Weights: Light, Regular, Bold, Black Mono companion: Zoho Puvi Mono
  • Semi-rounded details give warmth without losing professionalism
  • Strong tabular numerals for dashboard KPIs
  • Excellent legibility at small sizes (11-13px)
  • Wide language coverage for localized deployments
  • Open-source (Google Fonts) for broad licensing

Roboto

Alternate mode

Roboto is the second alternate typeface for Prism. A neo-grotesque sans-serif with mechanical skeleton and friendly curves, optimized for screen rendering.

  • Widest weight range (Light to Black) for maximum hierarchy control

Zoho Puvi usage rules

To keep Prism readable and scalable:

Rules

  • Default body text must use Zoho Puvi 14px
  • Dense table rows can use 13px
  • Never go below 12px
  • Use semibold before increasing size
  • Prefer sentence case
  • Keep table numerals aligned
  • Avoid ultra-light weights in dark mode
  • Use consistent heading weights across modules

Recommended weights

Do

Regular (400) for body text

Medium (500) for labels

SemiBold (600) for headings

Bold (700) for KPI emphasis only

Don't

Use Bold for all headings

Mix more than 2 weights per component

Use Light/Thin weights in dark mode

Increase font size when weight would suffice

Anti-patterns

Do

Use the defined type scale consistently

Add hierarchy through weight, not size

Match text styles from Figma exactly

Use monospace for all technical values

Don't

Create arbitrary font sizes (e.g., 15px, 17px)

Use ALL CAPS for body text

Nest more than 3 heading levels in a widget

Use Zoho Puvi for code snippets or logs

Principle

In Prism, Zoho Puvi is not just the brand font.

It is the reading speed layer that helps endpoint admins scan faster, compare data safely, and act confidently under pressure.
