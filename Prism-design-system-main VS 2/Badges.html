Badge

The Badge is a compact label component used to indicate status, category, or metadata. Badges appear in table cells, list items, navigation elements, and detail views to convey semantic meaning through color and style.

Content GuidelinesRecommended

Do
  • Use badges to communicate status at a glance.
  • Keep label text short (1-2 words).
  • Use consistent state colors across related views.
  • Use Subtle style when multiple badges appear together to avoid visual noise.
  • Use Intense style for the most important status in a view.
Don't
  • Don't use more than one badge per item in a list row.
  • Don't use badges for interactive/clickable elements (use Tags instead).
  • Don't mix Intense and Subtle styles within the same grouping.
  • Don't use long text strings that cause the badge to stretch excessively.

When and Where to Use

Understanding when to apply badges and when alternative components are more appropriate.

When to use

  • Use badges to communicate status at a glance in table cells, list items, navigation elements, and detail views.
  • Use for indicating metadata like version numbers, categories, or workflow stages.
  • Use when semantic color mapping helps users scan and prioritize items quickly.

When NOT to use

  • Not for interactive/clickable elements. Use a Tag component when the user can interact with the label.
  • Not for pure numeric values. Use a Counter component instead.
  • Not as a replacement for buttons or links.

Best practicesRecommended

Do
  • Use badges to communicate status at a glance.
  • Use Subtle style when multiple badges appear together to avoid visual noise.
  • Use Intense style for the most important status in a view.
Don't
  • Don't use badges for interactive elements (use Tags instead).
  • Don't mix Intense and Subtle styles within the same grouping.
  • Don't use long text strings that stretch the badge excessively.

RTL Support

  • When RTL = True, layout direction is mirrored -- the icon moves from leading to trailing position.
  • Text direction set to auto. All spacing and padding values are mirrored.
  • Shape (Pill vs Rounded) remains unchanged in RTL mode since border-radius is symmetrical.

Accessibility

Guidelines for ensuring badge status mappings are accessible to assistive technologies and adhere to contrast requirements.

Interactive Elements vs Badges

  • Badges are strictly non-interactive information indicators. They should not receive focus or be clickable.
  • If you require a clickable label, use the Tag component instead, which supports keyboard navigation and click events.
  • Use semantic HTML like <span> or <div> for badges to discourage screen readers from treating them as buttons.

Keyboard Interaction

Since badges are non-interactive, they do not participate in keyboard navigation.

KeyAction
TabBadge is skipped -- focus moves to the next focusable element.
Enter / SpaceNo action. Badges must not be interactive.
Note: If users need to interact with label-like elements, use the Tag component, which is keyboard-accessible and clickable.

ARIA Attributes

  • Since color carries meaning in badges (e.g. Red for Critical), ensure screen reader users receive the same context. You can append visually hidden text, like: <span class="sr-only">Status: Failed</span>.
  • If incorporating an icon purely for visual reinforcement, add aria-hidden="true" to the SVG element.

Focus State Design

Since badges are non-interactive, they do not receive focus. However, parent containers may.

  • Badges must never implement a focus ring since they are informational and not focusable.
  • If a badge is placed inside a focusable card or row, the parent element should display the focus ring -- not the badge itself.
  • Avoid tabindex on badge elements. A badge with tabindex is an anti-pattern.

RTL (Right-to-Left) Support

  • When RTL = On, the badge layout direction is mirrored automatically.
  • If an icon is present, it will predictably move from the leading to the trailing position.
  • All internal padding and spacing values mirror to preserve visual structure and logical reading order for screen readers.
