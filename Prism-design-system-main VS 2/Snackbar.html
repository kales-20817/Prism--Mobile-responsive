Toast Message

A brief, non-blocking notification that appears temporarily to inform users about the result of an action or system event. Supports four status types, two visual styles (Subtle and Filled), an optional auto-dismiss progress bar, CTA link, and close button.

Content GuidelinesRecommended

Do
  • Keep the title to one short line (8 words or fewer).
  • Use Success to confirm user actions positively.
  • Use Filled style for high-visibility or prominent alerts.
  • Show the Timeout Bar when duration is set.
Don't
  • Write a full paragraph as the title.
  • Use Info for everything — reserve statuses semantically.
  • Auto-dismiss Error toasts — users may not read them in time.
  • Let toasts pile up without a queue limit.

When and Where to Use

Toast messages provide brief, temporary feedback about an action or system event.

When to use

  • Confirm that a user-triggered action succeeded (e.g. "File saved", "Record deleted").
  • Surface a system event the user should be aware of but doesn't need to act on immediately.
  • When the notification is temporary and non-blocking — the user can continue working.
  • Use Info for neutral announcements, Success for confirmations, Warning for cautionary notices, Error for failures.

When NOT to use

  • Critical errors that block the user — use an Alert Banner or Inline Message.
  • Confirmations that require a decision — use a Dialog.
  • Persistent status the user needs to reference — use an Inline Message or Alert Banner.
  • Multiple simultaneous toasts beyond 3 — stack them but limit to 3 at a time maximum.

Placement Guidelines

Toast messages are typically positioned absolutely or fixed relative to the viewport.

Complete Do's and Don'ts

Guidelines for effective snackbar usage.

Do
  • Keep title short: One line, 8 words or fewer.
  • Use Success to confirm: Positively confirm user actions.
  • Use Filled for high-visibility: Prominent alerts that need attention.
  • Show CTA for reversible actions: Actions that need follow-up.
  • Use duration=0 for Errors: Errors need user acknowledgement.
  • Show Timeout Bar: When duration is set, it aids orientation.
  • Consistent positioning: Top-right or bottom-center consistently.
  • Animate in/out: Fade or slide transitions feel natural.
  • Limit stack to 3: Maximum 3 toasts simultaneously.
Don't
  • No paragraphs as titles: Keep titles concise and scannable.
  • Don't use Info for everything: Reserve statuses semantically.
  • Don't use Filled in dense pages: It competes visually with content.
  • Don't use CTA for unrelated nav: CTA should relate to the action.
  • Don't auto-dismiss Errors: Users may not read them in time.
  • Don't hide Timeout Bar: On auto-dismissing toasts it aids orientation.
  • Don't move toast position: Keep consistent between sessions.
  • Don't flash toasts instantly: No transition feels jarring.
  • Don't pile up without limit: Use a queue with max 3.

RTL Support

  • Set dir="rtl" on the toast container to mirror the layout.
  • When RTL is true, the child order becomes: Close → CTA → Text Content → Status Icon.
  • Arabic default text properties are provided for RTL variants.

Accessibility

Guidelines for ensuring snackbar notifications are accessible across assistive technologies.

ARIA Live Region

  • Polite announcements: Wrap the toast container in aria-live="polite" for Info, Success, and Warning toasts so screen readers announce new toasts without interrupting.
  • Assertive announcements: Use aria-live="assertive" for Error toasts that need immediate attention.
  • Role: Use role="status" for polite announcements or role="alert" for assertive (Error) toasts.

Keyboard Interaction

KeyAction
TabThe toast must be reachable via keyboard. Focus moves to the CTA or close button within the toast.
Enter / SpaceActivates the close button or CTA link when focused.
EscapeOptionally dismisses the focused toast.

Auto-dismiss Guidelines

  • Do not auto-dismiss Error toasts. Users need time to read and act on errors.
  • Minimum duration: Ensure duration ≥ 5000ms so users have time to read.
  • Pause on hover/focus: Pause the auto-dismiss timer when the user hovers over or focuses on the toast.

Close Button

  • Provide aria-label="Dismiss notification" on the close button for screen readers.
  • The close button must be activatable with Enter and Space keys.

Focus Management

  • Do not steal focus when a toast appears. The user's current task flow should not be disrupted.
  • If the toast contains a CTA, ensure it is reachable via Tab without disrupting the user's current flow.

Motion Preferences

  • Respect prefers-reduced-motion — replace slide/fade animations with instant show/hide.
  • The timeout bar countdown animation should also be disabled or replaced with a static indicator when reduced motion is preferred.

RTL Accessibility

  • Set dir="rtl" on the toast container for right-to-left languages.
  • The component's child order already mirrors for RTL; no additional ARIA attributes are needed for directionality.
