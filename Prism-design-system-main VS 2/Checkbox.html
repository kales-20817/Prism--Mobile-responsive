Checkbox

A Checkbox Group presents a set of options where users can select one or more items simultaneously. It wraps individual Checkbox items under a shared group label, optional help text, and coordinated validation states.

Content GuidelinesRecommended

Do
  • Use Left layout (default) for standard web forms.
  • Show a descriptive group label that frames the choices.
  • Use helpText to clarify constraints (e.g. “Select up to 3”).
  • Keep option labels concise (1–5 words).
Don't
  • Leave the group unlabelled.
  • Show only a red border without an error message.
  • Write full sentences as checkbox labels.
  • Pre-check options for commercial/legal consent.

When and Where to Use

A Checkbox Group lets users select none, one, or multiple options from a visible set of choices.

When to use

  • When users may select none, one, or multiple options from a list.
  • When all options should be visible simultaneously.
  • For preference, settings, or filter panels where multiple concurrent selections are valid.
  • When the number of options is small (2–8 items).

When NOT to use

  • Exactly one selection → use Radio Group.
  • Binary on/off with immediate effect → use Toggle.
  • Long list of options → use Multi-select Dropdown.
  • Single opt-in (e.g. “I agree”) → standalone Checkbox.

Alignment & Placement

Consistent placement makes it easier for users to locate and interact with checkbox groups.

  • Forms: Place checkbox groups vertically aligned with other form fields. Group label sits at the same level as input labels.
  • Settings panels: Left layout is preferred; each group occupies one row with the label flush-left and a header width of 240 px.
  • Mobile viewports: Switch to Top layout so options stack vertically with larger touch targets.
  • Dialogs: Checkbox groups in dialogs should be top-aligned with the content area. Keep groups compact (max 5–6 options).

Complete Do's and Don'ts

Visual and interaction guidelines for effective, accessible checkbox groups.

Do
  • Default layout: Use Left layout for standard web forms — most scannable.
  • Top for mobile: Use Top only when horizontal space is severely constrained.
  • Show help text: Clarify constraints like “Select up to 3”.
  • Error + message: Always pair Error state with a descriptive message.
  • Smart defaults: Pre-check options that apply to most users when appropriate.
Don't
  • No Top by default: Don’t use Top layout just because it feels familiar.
  • No unlabelled groups: Every checkbox group needs a descriptive label.
  • No red-only errors: Don’t show only a red border with no error message.
  • No sentence labels: Don’t write full sentences as checkbox labels.
  • No pre-checked consent: Never pre-check for commercial/legal consent.

Implementation Idea: Ember Hook Form

A common use case in Ember applications involves linking checkboxes to form changeset states.

<!-- MyApp/templates/components/permissions-form.hbs -->
<UemsCheckboxGroup
  @label="Permissions"
  @size="Medium"
  @value={{this.selectedPermissions}}
  @onChange={{fn this.updatePermissions}}
>
  <UemsCheckbox @value="read" @label="Read" />
  <UemsCheckbox @value="write" @label="Write" />
  <UemsCheckbox @value="delete" @label="Delete" />
</UemsCheckboxGroup>

RTL Support

  • When RTL = True, layout direction is mirrored — checkbox box swaps to the right side of the label.
  • Text direction set via dir="rtl" on a parent element. All spacing values mirror automatically. Controlled via RTL Layout variable collection.
  • Group label alignment and checkbox order mirror. Help text icon position swaps.

Accessibility

Guidelines for ensuring all checkbox group interactions are accessible across assistive technologies.

Keyboard Interaction

KeyAction
TabMoves focus between checkboxes in the group.
SpaceToggles the focused checkbox checked/unchecked.

ARIA Attributes & Screen Reader Guidance

Ensure checkbox group intent is explicitly communicated to all assistive technologies.

  • Use <fieldset> + <legend> (or role="group" + aria-labelledby) to group inputs.
  • Disabled State: Set disabled on <input type="checkbox">; do not use aria-disabled alone.
  • Error State: Associate help text via aria-describedby; pair with aria-invalid="true".
  • Screen Reader: Each option reads as: “[Label], checkbox, [checked/unchecked], [group label]”.
  • Indeterminate: Use aria-checked="mixed" on a parent/select-all checkbox when only some items are checked.
  • Don’t rely solely on color to communicate checkbox state — always pair with a visible checkmark or label.

Focus State Design

  • Focus ring: 2px solid using Border/Border-Accent-Focus.
  • Outline-offset: 2px from the checkbox box edge.
  • The checkbox box retains its current state styling inside the focus ring.
