Spacing

Create predictable, scannable, and dense-friendly endpoint management interfaces using a consistent spacing system.

Spacing in Prism is the structural rhythm behind dashboards, device tables, policy builders, patch workflows, alert feeds, and remediation drawers. A disciplined spacing system reduces visual noise, improves scan speed, and supports responsive density modes for IT admins.

8px base unit

Prism spacing is built on an 8px base unit.

Every spacing value must be a multiple or fraction of this unit to maintain consistency across modules.

8px
Base unit — all tokens are multiples

Spacing scale

Use only approved spacing tokens instead of raw pixel values.

Collection: UEMS Global Tokens · Scope: GAP

Additionally, spacing/0 is also available in the UEMS Theme Tokens collection with broader scopes (corner radius, width/height, gap, stroke, effect, opacity, font variations).

1. Group by task intent

Spacing should communicate operational relationships.

  • Filters stay visually close to result tables
  • Bulk actions stay close to selected row summaries
  • Deployment steps stay grouped by sequence
  • Device metadata fields use tighter spacing than dashboard cards

Rule: More related = less space. Less related = more space.

2. Design for density modes

Endpoint products need multiple density levels: Comfortable, Default, and Compact. Spacing tokens must scale predictably across density modes.

Example: Device table row

Comfortable
spacing/16
Default
spacing/12
Compact
spacing/8

Never create custom spacing outside the token scale.

3. Preserve table scan rhythm

Tables are the highest-usage pattern in endpoint products.

Table spacing rules

  • Cell padding: spacing/8 to spacing/12
  • Header to first row: spacing/8
  • Bulk toolbar to table: spacing/16
  • Table to pagination: spacing/16

Avoid oversized whitespace that slows row scanning.

4. Use progressive spacing hierarchy

Use larger spacing as the information hierarchy increases.

LevelToken
Inline label/valuespacing/4
Form field groupsspacing/12
Section blocksspacing/24
Workflow stepsspacing/32
Page modulesspacing/48

This creates visual rhythm across dashboards and workflows.

5. Align spacing with risk

High-risk actions need breathing room. Use larger spacing around:

  • Delete policy actions
  • Device isolation
  • Patch deployment confirmation
  • Bulk uninstall
  • Remote wipe
  • Restart workflows

Extra whitespace helps reduce accidental clicks.

Usage ranges

Small spacing: 0–8px

Use for compact UI and micro-alignment.

  • Icon + label gap
  • Status dot + text
  • Badge padding
  • Dense tree nodes
  • Checkbox and row label

Medium spacing: 12–24px

Use for standard component relationships. This is the most common range in endpoint UI.

  • Form groups
  • Card internal padding
  • Drawer sections
  • Filter rows
  • Toolbar grouping
  • Table header actions

Large spacing: 32–80px

Use for macro layout structure. Large spacing should define information zones, not decorative emptiness.

  • Dashboard widgets
  • Split page sections
  • Analytics panels
  • Hero sections
  • Empty states
  • Multi-step workflow pages

Negative spacing

Negative spacing should be rare and tokenized. Allowed only for:

  • Badge overlap on device avatars
  • Alert count overlays
  • Timeline connectors
  • Floating severity indicators
  • Notification stack offsets
spacing/negative-2
spacing/negative-4
spacing/negative-8

Never use arbitrary negative margins.

Endpoint-specific spacing patterns

Device inventory

  • Toolbar → table: spacing/16
  • Search → filters: spacing/12
  • Row icon → hostname: spacing/8

Patch deployment

  • Step header → body: spacing/16
  • Form sections: spacing/24
  • CTA zone → warning: spacing/12

Security dashboard

  • Threat cards: spacing/24
  • Chart → legend: spacing/8
  • Widget → widget: spacing/32

Tree navigation

  • Parent → child indent: spacing/16
  • Node icon → label: spacing/4

Spacing anti-patterns

Avoid these common enterprise UX failures:

  • Random 10px / 14px / 18px spacing
  • Different paddings for identical cards
  • Inconsistent drawer section gaps
  • Overspaced dense tables
  • Cramped confirmation dialogs
  • Uneven dashboard widget spacing
  • Tight destructive CTAs near safe actions
  • Tree indentation inconsistency

Accessibility and spacing

Spacing directly affects accessibility. Ensure spacing supports:

  • 200% zoom readability
  • Touch-friendly hit areas
  • Keyboard focus ring visibility
  • Reduced cognitive overload
  • Better low-vision grouping
  • Clear destructive-action separation

Whitespace is usability, not decoration.

Principle

In Prism, spacing defines operational clarity. Good spacing helps admins scan faster, act safer, and manage endpoints with confidence under pressure.
