Divider

A thin line used to visually separate content sections, list items, or layout regions. Supports horizontal and vertical orientations, three line styles, two thickness levels, four layout types including a centred label variant, and inset spacing options.

Content GuidelinesRecommended

Do
  • Keep With Text labels very short (1-3 words, e.g. "Or", "Today", "Continue").
  • Use Thin as the default thickness -- it is the least intrusive.
  • Use Solid as the default style; reach for Dashed/Dotted for softer boundaries.
Don't
  • Write full sentences as divider labels.
  • Use dividers as a substitute for vertical spacing -- use Spacing tokens instead.
  • Overuse dividers -- too many fragment a layout and increase visual noise.

States

The Divider is a non-interactive element with no hover, focus, or active states. Its appearance is entirely determined by the variant properties.

Dividers have no Boolean or Instance Swap properties -- all four axes are pure variant switches. The divider has no interactive states.

When and Where to Use

Dividers separate content sections, group related items, and define visual boundaries in layouts.

When to use

  • To visually separate distinct sections of a page, card, panel, or list.
  • To group related content -- a divider signals a boundary between two logical groups.
  • Use With Text to label a section transition (e.g. "Or", "Today", "More options").
  • Use Inset or Middle Inset inside lists where items have leading icons or avatars.
  • Use Vertical to separate items in a horizontal row (e.g. toolbar actions, breadcrumb segments).

When NOT to use

  • As a padding substitute -- use a Spacing token instead.
  • Inside dense data tables -- row borders serve the same purpose more efficiently.
  • Decoratively -- a divider communicates a semantic boundary; if there is no logical separation, omit it.
  • Overuse -- too many dividers fragment a layout and increase visual noise.

Alignment & Placement

Consistent placement ensures dividers are predictable and effective.

  • Full Width: Line spans the full available width or height. Use as the default for section separators.
  • Inset: 16 px indent on the leading edge only (left for horizontal, top for vertical). Use inside lists with leading icons.
  • Middle Inset: 16 px indent on both edges. Use when content has leading and trailing padding.
  • With Text: Line is split by a centred label with 8 px gaps. Use to label section transitions.
  • Vertical: Use between inline items in a horizontal row (toolbar actions, stat figures).

Complete Do's and Don'ts

Guidelines for effective divider usage.

Do
  • Use Thin for most separators: It is the default and least intrusive.
  • Use Solid as the default: Reach for Dashed/Dotted to signal softer or provisional boundaries.
  • Use Inset inside lists: Align the rule to the text column when items have leading icons or avatars.
  • Keep With Text labels short: 1-3 words (e.g. "Or", "Today", "Continue").
  • Let the Divider fill its container: Use CSS flex: 1 or width: 100%.
  • Use Vertical for inline items: Toolbar buttons, stats, and breadcrumb segments.
Don't
  • Default to Medium everywhere: Reserve it for major section breaks.
  • Use Dashed/Dotted without reason: Only use with a clear semantic purpose.
  • Use Full Width in icon lists: Use Inset instead when items have left-aligned icons or avatars.
  • Write full sentences as labels: Keep divider text extremely brief.
  • Use Vertical in vertical stacks: Use Horizontal there instead.
  • Hard-code fixed pixel width: Let horizontal dividers flow with their container.

RTL Support

  • Horizontal Full Width and With Text dividers are direction-neutral and render identically in LTR and RTL.
  • In RTL layouts, Inset indent moves from left to right automatically when using margin-inline-start instead of margin-left.
  • Vertical dividers remain unchanged in RTL as they are orientation-independent.
  • With Text label text direction follows the document dir attribute automatically.

Accessibility

Guidelines for ensuring divider elements are accessible across assistive technologies.

Keyboard Interaction

KeyAction
TabDividers are non-interactive and should never receive focus. Focus moves to the next interactive element.

Dividers are purely presentational. They do not receive keyboard focus and have no interactive behaviour.

ARIA Attributes & Screen Reader Guidance

  • Semantic element: Use <hr> for meaningful horizontal section breaks. It has an implicit role="separator".
  • Vertical dividers: Use <div role="separator" aria-orientation="vertical"> or <hr> with aria-orientation="vertical".
  • Decorative dividers: If purely visual, use <hr aria-hidden="true"> or a CSS pseudo-element so screen readers skip it.
  • With Text: The label text is already in the DOM -- no additional aria-label is needed. Ensure the surrounding wrapper has aria-hidden="true" if the text itself is the meaningful content.

Focus State Design

  • Dividers do not receive focus -- they are non-interactive presentational elements.
  • No focus ring or focus indicator is needed for dividers.

Color Contrast MinimumsIntentional

Border/Border-Tertiary (#E1E4EB) on white is low-contrast by design -- this is intentional for a subtle separator. Do not rely on the divider colour alone to communicate meaning.
